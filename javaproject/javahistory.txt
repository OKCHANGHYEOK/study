* 자바의 특징
 - 플랫폼으로부터 독립적이다.
 - 이를 가능하게 해주는건 JVM 때문

 * JVM
 - JAVA Virtual Machine 의 약자
 - 자바가 여러 운영체제에서 실행이 가능하도록 해주는 요소
 - 작동 방식 : 작성된 자바 코드가 JVM 에 컴파일되어 특정 영역에 저장이 된다.
 - 특정 영역
  - Heap 영역 : 참조 자료형 데이터가 저장된다. (배열, 객체 등)
  - Stack 영역 : 기본 자료형 데이터가 저장된다. (숫자, 문자, 부울)
  - Method 영역 : 컴파일된 자바 코드에 대한 정보가 저장되어 있음 (Meta Data)
 - 힙 영역에 저장된 참조 자료형 데이터의 값은 스택 영역으로부터 주소를 조회하여 찾아낸다.
  -> 배열이나 객체의 주소를 스택 영역이 갖고 있다.
 - 힙 vs 스택의 차이
  - 힙에 저장되는 참조 자료형의 데이터를 변경하면 가리키는 주소값이 바뀐다.
  - 스택에 저장되는 기본 자료형의 데이터를 변경하면 값 그 자체가 바뀐다.
